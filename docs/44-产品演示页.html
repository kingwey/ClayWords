想要一件"懂自己"的陶瓷，为什么这么难

标品千篇一律，定制门槛又太高。我们调研了 60 位有定制需求的用户和 12 家中小陶瓷工作室，两端都在喊累。

找不到，也撞款

差异化亮点 · 01

把陶瓷的"看不见"约束
写进生成流水线

陶瓷有壁厚、悬臂、底面稳定、烧成收缩这些看不见的硬约束——壁厚不到 3mm 烧裂、悬臂超 30° 塌陷、底面投影比小于 0.15 立不稳、烧成尺寸还要按材质收缩 8%–18%。

陶语用 numpy 把这四个真算法实现出来：Möller–Trumbore 批量射线相交求壁厚、面法线 vs 重力夹角识别悬臂、XY 投影面积比 + 重心高度判稳、按白瓷/紫砂/粗陶分独立阈值。失败自动修复 + 复检，**这是 text-to-3D 玩具变成生产工具的关键**。

RULE 01
壁厚 ≥ 3mm
射线相交批量采样，最薄处 < 阈值则触发外推修复
RULE 02
悬臂 ≤ 30°
面法线与重力夹角，朝上水平外伸面识别为悬臂
RULE 03
底面占比 ≥ 0.15
XY 投影面积比 + 重心高度比，头重脚轻惩罚
RULE 04
收缩补偿 1.08–1.18×

TRAE 实践

10 天 · 9 个 Phase · 130 条任务 · 一句话起步

没系统做过 3D 工程，靠 TRAE 把模糊想法拆成 9 Phase 130 任务的清单，一段段陪我写代码。最让我惊喜的不只是"会写代码"，而是它逼着我把"演示版"和"上线版"分开思考。

STEP 01

需求拆解与定位

"自然语言 → 可烧制陶瓷"的目标讲给 TRAE，它帮我拆出需求侧 / 供给侧两条痛点链，并锁定 200–800 元客单价区间作为切入市场。

STEP 02

任务清单与开发节奏

把整个项目拆成 9 个 Phase、130 条任务的清单，按"基建 → 数据 → 解析 → 3D → SSE → 派单 → 前端 → 加分项 → 彩排"的顺序推进，每条任务都附验收标准。

STEP 03

三路线技术评估

对比纯模板库 / 纯 text-to-3D / 检索增强混合三条路线，最终采用"A 模板派生（稳）+ B 生成式（多样）+ C 混合（创新）"+ 工艺校验兜底的架构。

STEP 04

核心模块编码

语义解析（LLM + JSON Schema）+ 工艺校验（4 真算法 + 15 单测）+ 派单评分（4 维加权）+ 12 节点状态机 + SSE 票据机制，全部在 TRAE 协助下落地。

STEP 05

前端三栏交互

三栏布局 + 6 釉色实时切换 + 方案版本树 + 派单可视化动画 + 印章工单弹窗 + 师傅接单 toast，全部基于 CSS variables token 体系。

STEP 06

演示鲁棒性

TRAE 主动设计三层兜底（在线案例池 / 离线 fixture / GPU 降级）+ 三个一键登录账号，确保现场断网或服务故障仍能完整演示。

STEP 07

代码审查

用 TRAE 的并行 agent 全面扫码，发现实体字段错位、路由顺序冲突、SVG 插值语法等隐藏 bug，逐一修复后给出"DEMO 完整度优先"的演示前定向加固清单。

STEP 08

本展示页生成

"我想要一个温润、有质感的陶瓷风格站点，要包含对话式设计台 + 派单可视化 + 工单印章" — TRAE 直接生成这份完整 HTML，配色、布局、交互、SVG 一次到位。
