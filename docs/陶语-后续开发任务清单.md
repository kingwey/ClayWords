## 使用说明

- **任务编号**：`Qx.y` 形式，`x` 为 Phase，`y` 为序号；编号即依赖顺序
- **状态**：`[ ]` 未开始 · `[~]` 进行中 · `[x]` 完成 · `[!]` 卡住 · `[-]` 取消（写理由）
- **优先级**：**P0** 公测必备 · **P1** 上线必备 · **P2** 加分项
- **责任人**：FE / BE / GPU / DSGN / OPS / DATA / QA / SEC
- **验收标准**：每条都可被验证，避免"自我感觉良好但跑不通"
