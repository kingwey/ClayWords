**2. Demo 创作思路**

- **灵感来源：** 朋友想给妈妈生日送一只"像她养的那只橘猫坐在月亮上"的陶瓷摆件，逛遍淘宝没找到，问了三家工作室都报价上千还要等两周打样。这件事让我意识到——自然语言到造型的"翻译"，正是大模型最擅长的事；而陶瓷行业，恰好缺这一环。
- **想解决的问题：**
  - **需求侧**：标品撞款，叙事性造型几乎搜不到；找工作室定制要 5–8 轮文字 + 草图来回，效果要等打样才能看到，沉没成本高
  - **供给侧**：中小工作室和非遗手艺人接单依赖熟人和私域，长尾需求接不住；师傅 60% 以上时间花在沟通上，真正拉坯烧窑的时间被严重挤压
- **为什么做这个方向：** 200–800 元客单价的"轻定制陶瓷"是目前严重缺供给的市场。AI 能解决的不只是"画一个图"，更关键的是把陶瓷的工艺约束写进生成流水线——壁厚不够 3mm 自动外推、悬臂超过 30° 自动加支撑、烧制收缩率按材质 1.08–1.18 倍预补偿。**这套"可烧制校验"才是把 text-to-3D 玩具变成生产工具的关键，也是陶语相对其他通用 AIGC 玩具的差异化所在**。我希望陶语既能让普通人 30 秒拥有一件"懂自己"的陶瓷，也能给景德镇、德化的师傅们装一条数字化的需求管道——这也是我加选 `社会公益` 标签的原因。

**4. TRAE 实践过程**

**开发流程：**

1. **需求拆解与定位**：把"自然语言 → 可烧制陶瓷"的目标讲给 TRAE，它帮我拆出需求侧 / 供给侧两条痛点链，并锁定 200–800 元客单价区间作为切入市场。
2. **任务清单与开发节奏**：TRAE 帮我把整个项目拆成 9 个 Phase、130 条任务的清单（[陶语开发任务清单](roadmap-v2.md)），按"基建 → 数据 → 解析 → 3D → SSE → 派单 → 前端 → 加分项 → 彩排"的顺序推进，每条任务都附验收标准。
3. **技术路线评估**：和 TRAE 一起对比纯模板库 / 纯 text-to-3D / 检索增强混合三条路线，最终采用"A 模板派生（稳）+ B 生成式（多样）+ C 混合（创新）"三路线并行 + 工艺校验兜底的架构。
4. **核心模块编码**：在 TRAE 协助下完成
   - 语义解析（LLM + JSON Schema 强制校验 + 黑名单 + LRU 缓存 + 101 条样本回归）
   - 工艺校验（4 个真算法 + 自动修复 + 15 pytest 单测）
   - 派单评分（4 维加权 + 4 级 tie-breaker + 中央工作室兜底）
   - 12 节点订单状态机 + 合法迁移表 + 状态日志
   - SSE 票据机制（一次性 ticket + 进度事件流）
5. **前端三栏交互**：TRAE 帮我落地三栏布局（聊天流 / 三方案卡片 / 3D 预览）+ 6 釉色实时切换 + 方案版本树 + 派单可视化动画 + 印章工单弹窗 + 师傅接单 toast，全部基于 CSS variables 的 token 体系，与产品展示页视觉一脉相承。
6. **演示鲁棒性**：TRAE 主动设计了三层 demo 兜底（在线案例池 / 离线 fixture / GPU 降级）和三个一键登录账号，确保评审现场断网或服务故障时仍能完整演示。
7. **代码审查**：用 TRAE 的并行 agent 全面扫码，发现多处实体字段错位、路由顺序冲突、SVG 插值语法等隐藏 bug 并逐一修复，最后用一份"DEMO 完整度优先"的修复清单做演示前 1 小时的定向加固。
