**1. Demo 简介**

* **是什么：** 一个对话式陶瓷定制 Web 应用——自然语言进、可烧制的陶瓷摆件出。

* **面向谁：**
  - 25-40 岁追求个性化家居 / 礼物的年轻人，新中式、治愈系、二次元偏好者
  - 想给亲友送有故事的礼物、但缺乏设计能力的送礼场景用户
  - 供给侧：景德镇、德化、宜兴等产区的中小陶瓷工作室与非遗手艺人

* **主要功能：**
  - **对话式设计台**：一句话描述意图，AI 自动解析造型 / 釉色 / 尺寸 / 情感意象，30 秒内并行生成 3 个可烧制的 3D 方案（A 模板派生最快 2 秒返回，B 生成式 + C 混合路线随后跟上，全部经过工艺校验）
  - **自然语言微调**：支持"把兔子耳朵再长一点点""加一抹胭脂红釉"这类局部指令，参考 SDEdit 思路对生成式 mesh 做选区局部重绘，所见即所得
  - **智能派单**：按工艺匹配度 / 产能 / 地域 / 信誉四维加权打分，自动匹配最合适的工作室；工艺校验通过的 GLB + 工艺参数包打成结构化工单直达师傅手中

**2. Demo 创作思路**

* **灵感来源：** 朋友想给妈妈生日送一只"像她养的那只橘猫坐在月亮上"的陶瓷摆件，逛遍淘宝没找到，问了三家工作室都报价上千还要等两周打样。这件事让我意识到——自然语言到造型的"翻译"，正是大模型最擅长的事；而陶瓷行业，恰好缺这一环。

* **想解决的问题：**
  - **需求侧**：标品撞款，叙事性造型几乎搜不到；找工作室定制要 5-8 轮文字 + 草图来回，效果要等打样才能看到，沉没成本高
  - **供给侧**：中小工作室和非遗手艺人接单依赖熟人和私域，长尾需求接不住；师傅 60% 以上时间花在沟通上，真正拉坯烧窑的时间被严重挤压

* **为什么做这个方向：** 200-800 元客单价的"轻定制陶瓷"是目前严重缺供给的市场。AI 能解决的不只是"画一个图"，更关键的是把陶瓷的工艺约束写进生成流水线——壁厚不够 3mm 自动外推、悬臂超过 30° 自动加支撑、烧制收缩率按材质 1.08-1.18 倍预补偿。这套"可烧制校验"是把 text-to-3D 玩具变成生产工具的关键。我希望陶语既能让普通人 30 秒拥有一件"懂自己"的陶瓷，也能给景德镇、德化的师傅们装一条数字化的需求管道——这也是我加选 `社会公益` 标签的原因。

**4. TRAE 实践过程**

**开发流程：**

1. **需求拆解与定位**：把"自然语言 → 可烧制陶瓷"的目标讲给 TRAE，它帮我拆出需求侧 / 供给侧两条痛点链，并锁定 200-800 元客单价区间作为切入市场
2. **技术路线评估**：和 TRAE 一起对比纯模板库 / 纯 text-to-3D / 检索增强混合三条路线，最终采用"A 模板派生（稳）+ B 生成式（多样）+ C 混合（创新）"三路线并行 + 工艺校验兜底的架构
3. **核心模块编码**：在 TRAE 协助下完成语义解析（LLM + JSON Schema 强制校验）、Hunyuan3D-2 推理接入、trimesh 工艺校验流水线、SSE 进度推送、派单算法这五块核心代码
4. **数据库与接口设计**：TRAE 帮我画 ER 图、写 Alembic 迁移、设计 OpenAPI 契约，重点解决了"下单锁定方案版本快照"和"phone_hash 加密 vs UNIQUE 索引"两个工程难题
5. **产品展示页生成**：告诉 TRAE "我要一个温润陶瓷质感的展示页，要包含对话式设计台演示"，它直接生成完整 HTML 文件，配色、布局、交互细节、AI 生成的产品参考图全部一次到位
6. **完整技术方案与演示方案产出**：TRAE 帮我整理出完整技术方案（含 15 个章节、风险预案、CI/CD、License 合规）和演示导向的精简版方案

1. **技术架构对话**：TRAE 输出三路线对比表与最终选型理由
