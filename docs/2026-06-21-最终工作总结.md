## 🎯 核心技术成果

### 数据层升级
- **SQLite → PostgreSQL 16** + pgvector 0.8.2
- **向量检索**: 1536 维 embedding + ivfflat 索引
- **Alembic 版本化**: 异步迁移系统
- **字段加密**: EncryptedStr 透明加密

### 任务队列系统
- **内存字典 → Redis Streams** 持久化队列
- **Consumer Group**: 多 Worker 并发消费
- **自动重试**: 3 次重试 + Dead Letter 队列
- **故障恢复**: Worker 重启自动认领 pending 任务

### 实时推送系统
- **内存 Queue → Redis Pub/Sub** 实时推送
- **Last-Event-ID**: 断线重连不丢事件
- **多实例**: 全局票据共享（TTL 60s）
- **keepalive**: 1 秒心跳保持连接

### 文件存储系统
- **MinIO 预签名 URL**: 客户端直传（1 小时有效）
- **MIME 验证**: 白名单 + 文件大小限制
- **安全扫描**: 状态机（pending → scanning → clean/quarantined）
- **Upload 表**: 完整的上传记录和状态追踪

### 工作室派单系统
- **入驻审核**: pending_review → approved/rejected
- **订单管理**: 接单/拒单/完成状态机
- **产能管理**: current_load 自动 +1/-1
- **派单评分**: 四维评分（工艺 35% + 产能 30% + 地理 15% + 评分 20%）

## 🔄 系统流程图

### 任务处理流程
```
Client → POST /sessions/{id}/messages
         ↓
   TaskService.create_task()
         ↓
   ┌─────┴────────────────────┐
   │                          │
PostgreSQL              Redis Stream
(持久化)                (队列: design.gen)
                              ↓
                        Worker XREADGROUP
                              ↓
                        process_task()
                              ↓
                    ┌─────────┴──────────┐
                    │                     │
              Pub/Sub               Event Stream
              (实时)                  (回放)
                    │                     │
                    ▼                     │
              SSE Client ◄────────────────┘
                        (Last-Event-ID 重连)
```

### 文件上传流程
```
Client → POST /uploads/init
         ↓
    MinIO presigned PUT URL
         ↓
    Client PUT → MinIO (直传)
         ↓
    POST /uploads/{id}/confirm
         ↓
    Upload.state = "clean"
         ↓
    public_url 可访问
```

### 工作室派单流程
```
订单创建 → 待派单
         ↓
   Dispatcher.dispatch_order()
         ↓
   四维评分排序
         ↓
   score >= 0.55 → 已派单
         ↓
   工作室接单 → 制作中
         ↓
   完成 → 已完成
   
   (拒单) → 待派单 (重新派单)
```

## 💡 关键决策

### 1. 跳过 Phase Q3
**原因**: 3D 模型生成工作量大（12 天），需要 GPU 环境和设计师资源  
**策略**: 先完成业务闭环，Phase Q3 可并行或后续补充

### 2. 采用 Redis Streams
**原因**: 比 RabbitMQ 轻量，比 Kafka 简单，自带持久化和重放  
**收益**: 开发效率高，运维成本低

### 3. MinIO 预签名上传
**原因**: 减轻服务器带宽压力，提升上传体验  
**收益**: 服务器只处理元数据，文件直传对象存储

### 4. 四维评分派单
**原因**: 平衡工艺、产能、距离、评分，避免单一维度偏差  
**收益**: 派单质量高，工作室负载均衡
