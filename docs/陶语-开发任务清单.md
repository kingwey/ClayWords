## 使用说明

- **任务编号**：`Px.y` 形式，`x` 为 Phase，`y` 为任务序号；编号即依赖序，原则上后置任务依赖前置完成
- **状态标记**：`[ ]` 未开始 · `[~]` 进行中 · `[x]` 完成 · `[!]` 卡住需协助 · `[-]` 取消（写明原因）
- **优先级**：**P0** 演示必备 · **P1** 加分项 · **P2** 不做（占位记录）
- **责任人**：FE / BE / GPU / DSGN / OPS / DATA（前端 / 后端 / GPU 工程 / 设计 / 运维 / 数据）
- **验收标准**：每条任务都附"如何证明完成了"——避免自我感觉良好但跑不通
- **参考章节**：链接到设计文档对应位置，写代码前先读那一节
