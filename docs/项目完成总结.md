```
    ┌────▼───┐ ┌───▼────┐ ┌──▼─────┐

    └────┬───┘ └───┬────┘ └────────┘
         │         │

    │Alembic │ │ Worker │
    │Migration│ │Consumer│
    └────────┘ └────────┘
```

### 数据流

```
用户请求
  ↓
[LoggingMiddleware: request_id 注入]
  ↓
[PrometheusMiddleware: 指标采集]
  ↓
[RateLimitMiddleware: 速率限制]
  ↓
[JWT 认证]
  ↓
FastAPI 路由
  ↓
SQLAlchemy ORM (异步)
  ↓
PostgreSQL 16 + pgvector
```

### 任务处理

```
用户提交任务
  ↓
TaskService.create_task()
  ↓
双写: PostgreSQL + Redis Stream
  ↓
Worker Consumer Group 并发消费
  ↓
处理任务 (重试 3 次)
  ↓
Redis Pub/Sub 发送事件
  ↓
SSE 推送到客户端
```

## 💡 关键技术亮点

### 1. 向量检索优化

```sql
CREATE INDEX ON design_templates 
USING ivfflat (embedding vector_cosine_ops) 
WITH (lists = 100);
```

**优势**: 1536 维向量检索 < 10ms

### 2. 双写策略保证可靠性

```python
# 1. PostgreSQL 持久化
task = await db.add(task)

# 2. Redis Stream 入队
await redis.xadd("tasks_stream", {...})
```

**优势**: 数据不丢失 + 高性能

### 3. SSE 断线重连

```python
# 客户端发送 Last-Event-ID
last_id = request.headers.get("Last-Event-ID")

# 服务端回放历史
if last_id:
    history = await redis.xrange("events", last_id)
    for event in history:
        yield event
```

**优势**: 30 秒内断网不丢消息

### 4. 幂等性键防重

```python
idempotency_key = f"payment_callback_{trade_no}"
if await redis.exists(idempotency_key):
    return "success"  # 已处理

# 处理支付逻辑
await redis.setex(idempotency_key, 86400, "processed")
```

**优势**: 支付回调防重放

### 5. 多层防护体系

```
用户输入 → Pydantic 验证 → 速率限制 → Prompt 防护 
→ JWT 认证 → 参数化查询 → 字段加密 → 数据库
```

**优势**: 纵深防御
