### ✅ 任务 1: 全局 HTTP 客户端连接池 (P0-2)

**问题描述**: 每次调用外部 API 都创建新的 `httpx.AsyncClient`，高并发下会耗尽文件描述符。

**执行内容**:

1. **创建 HTTP 客户端管理器** - [backend/app/core/http_client.py](backend/app/core/http_client.py)
   ```python
   class HTTPClientManager:
       """单例 HTTP 客户端管理器，配置连接池"""
       
       @classmethod
       async def get_client(cls) -> httpx.AsyncClient:
           if cls._instance is None or cls._instance.is_closed:
               cls._instance = httpx.AsyncClient(
                   timeout=httpx.Timeout(30.0, connect=10.0),
                   limits=httpx.Limits(
                       max_connections=100,           # 最大连接数
                       max_keepalive_connections=20,  # Keep-alive 池大小
                       keepalive_expiry=30.0,        # Keep-alive 过期时间
                   ),
                   follow_redirects=True,
                   http2=True,  # 启用 HTTP/2
               )
           return cls._instance
   ```

2. **注册应用关闭钩子** - [backend/app/main.py](backend/app/main.py:45-48)
   ```python
   @asynccontextmanager
   async def lifespan(app: FastAPI):
       # ... 启动逻辑 ...
       yield
       await shutdown_http_client()  # 新增：关闭 HTTP 客户端
       await redis_client.disconnect()
   ```

3. **修改 Hunyuan3D 客户端** - [backend/app/services/hunyuan3d/client.py](backend/app/services/hunyuan3d/client.py)
   - ❌ 修改前: `async with httpx.AsyncClient(timeout=30.0) as client:`
   - ✅ 修改后: `client = await get_http_client()`

**预期效果**:
- 减少 TCP 连接创建开销：从每次请求创建 → 连接池复用
- 防止文件描述符耗尽：限制最大连接数 100
- 性能提升：Keep-alive 连接复用，HTTP/2 多路复用

2. **使用数据库唯一约束防止竞态** - [backend/app/api/payments.py:162-188](backend/app/api/payments.py#L162-L188)
   
   **修改前** (有竞态窗口):
   ```python
   # 步骤 1: 查询幂等性键
   existing = await db.execute(select(IdempotencyKey).where(...))
   if existing:
       return "success"
   
   # 步骤 2: 写入幂等性键
   session.add(idem_key)
   # ⚠️ 竞态窗口：两个并发回调都通过了步骤 1
   ```

   **修改后** (原子操作):
   ```python
   # 直接写入，让数据库主键唯一约束保证原子性
   session.add(idem_key)
   try:
       await session.flush()  # 强制刷新，触发唯一约束检查
   except IntegrityError:
       # 重复回调，已被其他请求处理
       await session.rollback()
       logger.info("payment_callback_duplicate", trade_no=trade_no)
       return "success"
   ```

3. **添加日志记录** - [backend/app/api/payments.py:10](backend/app/api/payments.py#L10)

   ```

**预期效果**:
- 彻底防止支付回调重复处理
- 避免重复扣款、容量重复占用
- 数据库层面保证原子性，无需应用层分布式锁

   async def release_studio_capacity(db: AsyncSession, studio_id: str):
       """原子减少工作室容量，避免并发问题"""
       result = await db.execute(
           update(Studio)
           .where(Studio.studio_id == studio_id)
           .where(Studio.current_load > 0)  # 防止负数
           .values(current_load=Studio.current_load - 1)
       )

       if result.rowcount > 0:
           metrics.set_studio_load(studio_id, studio.current_load)
   ```

### 📋 任务 5: Worker 结果双写 PostgreSQL (P0-5)

**问题描述**: Worker 任务结果只存 Redis (TTL 1 小时)，Redis 重启或内存淘汰会导致结果丢失。

**影响**:
- 用户刷新页面查询任务结果时，返回 404
- 无法审计历史任务记录
- 生产故障恢复困难

**执行方案**:

1. **修改 Worker 任务完成逻辑** - [worker/main.py:136-153](worker/main.py#L136-L153)
   ```python
   # 当前：只写 Redis
   await redis_client.set(
       f"task:{task_id}:result",
       json.dumps(result),
       ex=3600
   )
   
   # 修改后：双写 PostgreSQL + Redis
   # 1. 写入 PostgreSQL (持久化)
   await db.execute(
       update(Task)
       .where(Task.task_id == task_id)
       .values(
           state="completed",
           result=result,  # JSONB 字段
           completed_at=utcnow()
       )
   )
   await db.commit()
   
   # 2. 写入 Redis (缓存层，1 小时过期)
   await redis_client.set(
       f"task:{task_id}:result",
       json.dumps(result),
       ex=3600
   )
   ```

2. **修改任务查询逻辑** - [backend/app/services/tasks/task_service.py](backend/app/services/tasks/task_service.py)
   ```python
   async def get_task_result(self, task_id: str):
       # 1. 先查 Redis 缓存
       cached = await redis.get(f"task:{task_id}:result")
       if cached:
           return json.loads(cached)
       
       # 2. Redis miss，回源 PostgreSQL
       stmt = select(Task).where(Task.task_id == task_id)
       result = await db.execute(stmt)
       task = result.scalar_one_or_none()
       
       if not task or task.state != "completed":
           return None
       
       # 3. 回填 Redis 缓存
       await redis.set(
           f"task:{task_id}:result",
           json.dumps(task.result),
           ex=3600
       )
       
       return task.result
   ```

3. **数据库迁移** (如果 `tasks` 表缺少 `result` 字段)
   ```bash
   cd backend
   alembic revision --autogenerate -m "add_result_to_tasks"
   alembic upgrade head
   ```
